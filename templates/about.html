About Conference User Creation

This web frontend provides a user-friendly interface for generating PowerShell commands to create and manage conference workshop user accounts in Azure/Entra ID environments.

The underlying PowerShell scripts automate the process of setting up multiple user accounts, groups, and resources for conferences, workshops, and training events, making it easy to provision standardized environments for participants.

Features
  • Automatic tenant discovery
  • Standardized naming patterns
  • Entra ID group management
  • Azure resource group creation
  • Flexible configuration options
  • Unique password generation
  • Excel export capability
  • Dry run preview mode
  • Comprehensive error handling
  • Cleanup support
How It Works
  1. Fill the web form with your conference parameters
  2. Generate the PowerShell command with proper syntax
  3. Copy the command to your clipboard
  4. Run the command in PowerShell with appropriate permissions
  5. Users and resources are created automatically

The web interface acts as a command generator - it doesn't directly modify your Azure environment but creates the exact PowerShell commands you need.

Prerequisites
PowerShell Modules

Install the required Microsoft Graph PowerShell modules:

Install-Module Microsoft.Graph.Authentication -Force
Install-Module Microsoft.Graph.Users -Force
Install-Module Microsoft.Graph.Groups -Force
Install-Module Microsoft.Graph.Identity.DirectoryManagement -Force
For Azure Resource Groups
Install-Module Az.Accounts -Force
Install-Module Az.Resources -Force
For Excel Output (Optional)
Install-Module ImportExcel -Force
Azure Permissions

The user running the script must have sufficient permissions in the Azure tenant:

Permission Purpose
User.ReadWrite.All To create and manage users
Directory.Read.All To read tenant information
Group.ReadWrite.All To create and manage groups
Contributor or Owner For Azure resource group creation
Security Considerations
  • Password Management: Unique random passwords are generated for each user when no password is specified
  • Permissions: High-level permissions are required - ensure appropriate access controls
  • Audit Trail: User creation activities are logged in Azure AD audit logs
  • Access Control: Created users inherit default tenant permissions - review as needed
  • Cleanup: Always remove test users and resources when no longer needed
Important Limitations
  • MSA Personal Accounts: Cannot be used with Microsoft Account (MSA) personal accounts
  • Organizational Account Required: Must use work or school account with Azure AD/Entra ID access
  • Network Requirements: Requires internet connectivity to Azure services
  • PowerShell Version: Requires PowerShell 5.1 or higher
